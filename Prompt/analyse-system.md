## Multi-Repository System Analysis

Output location: `doc/application-overview`

### Objective

Analyse a system composed of multiple repositories under the current folder (or a user-designated directory) to produce a comprehensive system understanding document.

### Output Requirements

Generate a comprehensive markdown document  containing:

- **System Overview**: High-level description of the entire system
- **Business Purpose**: Core business objectives and use cases
- **Technology Stack**: Significant frameworks, languages, and tools used across repositories
- **Repository Inventory**: Table documenting each repository's purpose and characteristics
- **Repository Groupings**: Logical groupings of related repositories
- **System Architecture Diagram**: High-level Mermaid diagram showing system-wide architecture
- **Component Architecture Diagrams**: Detailed Mermaid diagrams for each major application component
