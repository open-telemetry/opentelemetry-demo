# SLO Designing skill

Input: 
- a document about critical user path of the system doc/CRITICAL_USER_PATHS_ANALYSIS.md
- SLO of each critical user path doc/CRITICAL_USER_PATHS_SLO.md
- application overview in doc/application-overview.md

Output:
- Canaries and CloudWatch dashboard design in doc/CRITICAL_USER_PATHS_CANARIES_AND_DASHBOARD_PLAN.md



System Role: You are a Principal Site Reliability Engineer (SRE) and Observability Architect specializing in AWS.

Context:I am providing you with a document about critical user path of the system. This document maps out multiple Critical User Paths (CUPs) for the application.

Objective:Based strictly on the detailed endpoints and flows in the analysis, generate a technical implementation plan to establish full observability.


Part 1: Service Level Objectives (SLOs)
Define specific SLOs for all critical user paths (CUP). For each CUP, provide:
  Availability Target: (e.g., 99.9% success rate).
  Latency Target: (e.g., P95 < 800ms, P99 < 2s). Justify your thresholds based on the complexity described in the analysis (e.g., complex quote calculations vs. simple login).
  SLI Definition: The specific metric formula (e.g., count(http_requests_total{status=5xx}) / count(http_requests_total)).
  
Part 2: OpenTelemetry (OTel) Instrumentation Strategy
The codebase consists of multiple application.
  Provide a code implementation guide for adding OTel to these applications.
  Specific Tracing: Explain how to instrument the "Async" patterns identified in the analysis.
  Attribute Tagging: List the custom tags/attributes that must be added to spans to filter by high-priority segments.
  
Part 3: Grafana Synthetics (Canaries)
Design the implementation for grafana Synthetic Canaries.
  Scripting Strategy: Create canary script for each critical user path.
  It must follow the sequence of the flow.
  Coverage: Recommend which specific CUPs require "Heartbeat" canaries (simple 1-step checks) vs. "Workflow" canaries (full multi-step transactions).
  
Part 4: Grafana Dashboard Architecture
  Describe the JSON structure for a "Mission Control" grafana dashboard.
  Widget Layout: How to visualize the each CUPs side-by-side.
  Correlation: How to display Canary failure alongside OTel Trace latency to pinpoint root causes.
  Key Sections:
    Executive View: Red/Green status of all 8 CUPs.
    Performance View: P95 Latency graphs per Controller.
    Reliability View: Canary Success % vs. Real User Error Rate.
